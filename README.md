# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

# Данные

## Интерфейсы и типы данных

### IProduct

Интерфейс описывает товар, получаемый с сервера и отображаемый в каталоге.

| Поле          | Тип              | Описание                                                          |
| ------------- | ---------------- | ----------------------------------------------------------------- |
| `id`          | `string`         | Уникальный идентификатор товара                                   |
| `description` | `string`         | Подробное описание товара                                         |
| `image`       | `string`         | URL изображения товара                                            |
| `title`       | `string`         | Название товара                                                   |
| `category`    | `string`         | Категория товара                                                  |
| `price`       | `number \| null` | Цена товара. Может быть `null`, если товар недоступен для покупки |

---

### TPayment

Тип данных, описывающий доступные способы оплаты заказа.

```typescript
type TPayment = 'card' | 'cash' | '';
```

| Значение | Описание                 |
| -------- | ------------------------ |
| `'card'` | Оплата банковской картой |
| `'cash'` | Оплата наличными         |
| `''`     | Способ оплаты не выбран  |

---

### IBuyer

Интерфейс содержит данные покупателя, необходимые для оформления заказа.

| Поле      | Тип        | Описание                |
| --------- | ---------- | ----------------------- |
| `payment` | `TPayment` | Выбранный способ оплаты |
| `email`   | `string`   | Адрес электронной почты |
| `phone`   | `string`   | Контактный телефон      |
| `address` | `string`   | Адрес доставки          |

---

### IOrder

Интерфейс описывает заказ, который отправляется на сервер.

Наследуется от интерфейса `IBuyer`.

| Поле    | Тип        | Описание                       |
| ------- | ---------- | ------------------------------ |
| `items` | `string[]` | Массив идентификаторов товаров |
| `total` | `number`   | Общая стоимость заказа         |

---

### IOrderResponse

Интерфейс описывает успешный ответ сервера после оформления заказа.

| Поле    | Тип      | Описание                        |
| ------- | -------- | ------------------------------- |
| `id`    | `string` | Идентификатор созданного заказа |
| `total` | `number` | Итоговая стоимость заказа       |

---

### ICatalogueResponse

Интерфейс описывает ответ сервера при запросе каталога товаров.

| Поле    | Тип          | Описание                 |
| ------- | ------------ | ------------------------ |
| `items` | `IProduct[]` | Список товаров           |
| `total` | `number`     | Общее количество товаров |

---

## Модели данных

### Класс Catalogue

Класс отвечает за хранение каталога товаров и выбранного пользователем товара. Используется для сохранения полученных с сервера товаров, поиска товара по идентификатору и хранения текущего выбранного товара.

#### Конструктор

```typescript
constructor()
```

Конструктор не принимает параметров. При создании экземпляра создаёт пустой каталог товаров и сбрасывает выбранный товар.

#### Поля класса

| Поле           | Тип                | Описание                                     |
| -------------- | ------------------ | -------------------------------------------- |
| `items`        | `IProduct[]`       | Массив товаров каталога                      |
| `selectedItem` | `IProduct \| null` | Товар, выбранный пользователем для просмотра |

#### Методы класса

##### save(items: IProduct[]): void

Сохраняет список товаров в каталог.

**Параметры:**

* `items: IProduct[]` — массив товаров.

##### getItems(): IProduct[]

Возвращает полный список товаров каталога.

##### getItemById(id: string): IProduct | undefined

Ищет товар по идентификатору.

**Параметры:**

* `id: string` — идентификатор товара.

**Возвращает:**

* `IProduct` — найденный товар;
* `undefined` — если товар не найден.

##### saveSelectedItem(selectedItem: IProduct): void

Сохраняет выбранный пользователем товар.

**Параметры:**

* `selectedItem: IProduct` — выбранный товар.

##### getSelectedItem(): IProduct | null

Возвращает текущий выбранный товар.

---

### Класс Cart

Класс отвечает за управление корзиной покупок. Позволяет добавлять и удалять товары, получать список товаров в корзине, вычислять общую стоимость заказа и количество выбранных товаров.

#### Конструктор

```typescript
constructor()
```

Конструктор не принимает параметров. При создании экземпляра создаёт пустую корзину.

#### Поля класса

| Поле            | Тип          | Описание                              |
| --------------- | ------------ | ------------------------------------- |
| `selectedItems` | `IProduct[]` | Массив товаров, добавленных в корзину |

#### Методы класса

##### getSelectedItems(): IProduct[]

Возвращает список товаров, находящихся в корзине.

##### addItem(item: IProduct): void

Добавляет товар в корзину.

**Параметры:**

* `item: IProduct` — товар для добавления.

##### deleteItem(item: IProduct): void

Удаляет товар из корзины.

**Параметры:**

* `item: IProduct` — товар для удаления.

##### clear(): void

Полностью очищает корзину.

##### getTotalPrice(): number

Вычисляет общую стоимость всех товаров в корзине.

**Возвращает:**

* `number` — сумма всех цен товаров.

##### getNumberOfItems(): number

Возвращает количество товаров в корзине.

##### itemExist(id: string): boolean

Проверяет наличие товара в корзине.

**Параметры:**

* `id: string` — идентификатор товара.

**Возвращает:**

* `true` — если товар найден;
* `false` — если товар отсутствует.

---

### Класс Buyer

Класс отвечает за хранение данных покупателя и их валидацию перед оформлением заказа.

#### Конструктор

Явный конструктор отсутствует. Все поля инициализируются значениями по умолчанию:

```typescript
payment = ''
email = ''
phone = ''
address = ''
```

#### Поля класса

| Поле      | Тип        | Описание                           |
| --------- | ---------- | ---------------------------------- |
| `payment` | `TPayment` | Выбранный способ оплаты            |
| `email`   | `string`   | Адрес электронной почты покупателя |
| `phone`   | `string`   | Контактный номер телефона          |
| `address` | `string`   | Адрес доставки                     |

#### Методы класса

##### setPayment(payment: TPayment): void

Устанавливает способ оплаты.

##### setEmail(email: string): void

Сохраняет адрес электронной почты покупателя.

##### setPhone(phone: string): void

Сохраняет номер телефона покупателя.

##### setAddress(address: string): void

Сохраняет адрес доставки.

##### getData(): IBuyer

Возвращает объект с данными покупателя.

##### clear(): void

Очищает все сохранённые данные покупателя.

##### validatePaymentAndAddress(): { payment?: string; address?: string }

Проверяет корректность заполнения способа оплаты и адреса доставки.

Возвращает объект с сообщениями об ошибках.

##### validateContactInfo(): { email?: string; phone?: string }

Проверяет заполнение контактной информации.

Возвращает объект с сообщениями об ошибках.

##### validation(): { payment?: string; address?: string; email?: string; phone?: string }

Выполняет полную проверку данных покупателя.

Возвращает объект, содержащий сообщения обо всех найденных ошибках.
